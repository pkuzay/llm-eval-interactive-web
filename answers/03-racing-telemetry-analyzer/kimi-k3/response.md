## 使用方式

```bash
python RAC遥测多圈对比分析工具.py
```

打开后点 **「启动模拟源」→「启动接收」→「● 开始记录」** 即可看到实时绿色曲线；模拟车跑完一圈自动存 CSV，再点 **「载入 Lap A / Lap B」** 选择两圈文件即进入对比模式，鼠标悬停显示十字线和两圈差值。

## 四大技术支柱的实现要点

**1. UDP 接收器** — `TelemetryReceiver` 独立线程，socket 设 0.2s 超时的伪非阻塞模式（既能及时响应停止信号，又绝不阻塞 UI），解析结果经 `queue.Queue(maxsize=20000)` 推送到主线程，队列满时丢帧计数而非阻塞。同时兼容 **JSON 文本帧**和 **struct `<5d` 二进制帧**，带合理性校验（速度/距离越界、NaN 直接丢弃）。

**2. 距离对齐重采样** — 这是赛车遥测的核心：`build_common_grid()` 取两圈距离的**交集区间**，以 1 米步长生成公共网格，再用 `numpy.interp` 把两圈的 speed/throttle/brake 插值到同一空间坐标系。`LapData.load_samples()` 会先排序并剔除距离重复/倒退点（插值要求 X 严格递增）。这样快圈慢圈在横轴上严格对齐，"哪里刹车早了、哪里给油慢了"一目了然。

**3. 记录与持久化** — 主循环每 50ms 排空队列，检测 `distance` 突降 >50m 判定为过线/重置，自动把当前圈写入 `telemetry_logs/lap_YYYYMMDD_HHMMSS.csv`（UTF-8-BOM，表头即五个字段），也可手动存圈；退出时若有未保存数据会弹窗询问。

**4. 交互 UI** — tkinter + matplotlib 双图垂直堆叠（Speed/Pedals vs Distance，共享 X 轴），蓝=Lap A、红=Lap B、绿=实时圈；十字线在两张图联动，悬停框实时显示该距离点上两圈的速度差（Δ km/h）和油门/刹车差。实时模式用 600m 滑动窗口跟随，避免大数据量卡顿。
