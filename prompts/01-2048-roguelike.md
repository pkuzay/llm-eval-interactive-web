# 任务目标
请发挥你的创意上限，设计并编写一个单文件网页版（HTML + CSS + JS）的【2048 融合机制 + Roguelike 元素】的游戏。你可以自由决定游戏的主题、背景设定和整体视觉风格。

# 核心评测维度 (你将在这两方面接受严格评估)

## 维度 A：前端动效与流畅度 (权重 50%)
我们极其看重游戏的“手感”与“视觉解压感（Juice）”。请在代码中展现你最顶尖的前端优化与动画技术：
- **无缝滑移与融合：** 卡牌/方块的滑动、碰撞、合并必须极为丝滑，绝对不能出现闪烁、生硬位移或卡顿。合并时需要有明显的弹性缩放、微震动等物理反馈。
- **动态交互响应：** 用户的每一次滑动操作，场景、UI 或特效都应该有即时的、符合物理直觉的动态反馈。
- **技术栈自由：** 你可以自由选择使用纯 CSS 3D、HTML5 Canvas、或是通过 CDN 引入 Three.js / PixiJS / GSAP 等任何能实现极致动效的库。

## 维度 B：Roguelike 流程理解 (权重 50%)
请证明你对 Roguelike 游戏精髓的理解，构建一个完整的单局游玩生命周期，必须包含以下闭环：
- **局内成长与 Build 构建：** 玩家在合并方块或达成特定条件时，能够获得不同流派的“遗物/被动技能/卡牌增强”，这些强化能切实改变后续游戏的操作或结算逻辑。
- **动态生成的挑战：** 随着游戏推进（层数/关卡增加），玩家会面临具有差异化的敌人或障碍挑战，游戏难度曲线需合理递增。
- **死亡与重来：** 包含明确的资源消耗、血量或失败机制。失败后提供清晰的结算与重新开始流程，且每一局的体验应具有随机性。

# 技术与输出要求
- 所有的代码（HTML、CSS、JS）必须写在同一个 `.html` 文件中。
- 代码必须完全闭合且可运行，不允许出现任何 `// TODO` 或未完成的占位符。
- 允许通过 CDN 引入外部开源库（请保证链接有效）。
